# ViewTune

> 영상 속도와 화면을, 한 키로 맞추다.

ViewTune은 YouTube를 포함한 웹의 HTML5 비디오를 키보드로 제어하는 가벼운 Chrome 확장입니다. 평소에는 페이지 위에 버튼을 띄우지 않고, 조작이 성공했을 때만 짧은 피드백을 표시합니다.

Chrome 114 이상이 필요합니다. 창 맞춤이 사용하는 표준 Popover top layer가 Chrome 114부터 제공되기 때문입니다.

## 핵심 기능

- 현재 보고 있는 비디오를 0.5× 단위로 조절하고 1×로 초기화
- 사용자가 정한 목표 속도로 한 번에 전환(기본 2×)
- 브라우저 탭 안에서 플레이어와 컨트롤을 함께 최대화
- 현재 플레이어·전체화면 크기는 유지하고 영상만 확대해 빈 영역 없이 채우기
- 한/영 입력 상태와 무관한 물리 키 위치 기반 단축키
- 팝업의 버튼 조작과 별도 설정 화면의 단축키 재지정

## 기본 단축키

| 동작 | 키 위치 | 설명 |
| --- | --- | --- |
| 21:9 확대 토글 | `B` | 넓은 프레임 안에서 영상만 확대, 다시 누르면 복원 |
| 창 맞춤 토글 | `V` | 일반·영화관 모드를 탭 화면에 맞춤, 다시 누르거나 `Esc`로 복원 |
| 목표 속도 적용 | `G` | 설정한 속도로 바로 전환, 기본값 2× |
| 속도 0.5× 낮춤 | `[` | 왼쪽 대괄호 |
| 속도 0.5× 높임 | `]` | 오른쪽 대괄호 |
| 속도 1× 초기화 | `R` | Reset, 현재 비디오만 초기화 |

YouTube의 음소거·다음 영상·검색·속도 기본키와 겹치는 `N`, `M`, `<`, `>`, `/`는 기본값에서 제외했습니다. 자막 창 불투명도에 쓰이는 `W`와 미니 플레이어 `I`도 피하고, 화면·속도 모드를 왼손으로 오가기 쉬운 `B`, `V`, `G`에 모았습니다. 단축키는 글자값이 아니라 `KeyboardEvent.code`를 비교하므로 한글 IME 상태에서도 같은 물리 키 위치에서 동작합니다. 입력창, 검색창, 댓글창, `contenteditable` 영역과 제어할 영상이 없는 페이지에서는 단축키를 가로채지 않습니다.

## 화면 모드의 의미

- **창 맞춤(V):** 운영체제 전체 화면이나 F11이 아닙니다. YouTube 일반 모드와 `T` 영화관 모드, 라프텔처럼 컨트롤이 자동으로 숨겨지는 플레이어에서도 확인된 플레이어 묶음을 현재 브라우저 탭의 보이는 영역에 맞춥니다. 영상은 원본 비율을 유지해 전부 보이며, `V`를 다시 누르거나 `Esc`를 누르면 복원됩니다.
- **21:9 확대(B):** 현재 플레이어 또는 전체화면의 크기와 상태는 바꾸지 않습니다. 현재 프레임이 원본 영상보다 충분히 넓을 때만 영상 표시 영역을 `cover`로 확대해 빈 공간을 채우며, 위아래 일부가 잘릴 수 있습니다. 21:9 모니터에서 YouTube·Netflix 전체화면을 사용한 뒤 `B`를 누르는 경우가 대표적입니다. 일반적인 16:9 영상과 16:9 플레이어처럼 확대할 여백이 없으면 좌표를 건드리지 않고 안내만 표시합니다.

두 기능은 독립적입니다. V와 B를 함께 켤 수 있고, 어느 하나를 꺼도 다른 기능은 유지됩니다. B를 켠 뒤 화면이 잠시 16:9가 되면 확대 표시만 쉬었다가 다시 넓어졌을 때 자동으로 적용됩니다.

V는 사이트의 특정 ID를 정답으로 간주하지 않습니다. 현재 영상과 겹치는 플레이어 표면, 실제로 보이는 컨트롤, 같은 프레임을 나타내는 가까운 조상이라는 기능·기하 조건을 확인한 뒤 그 묶음을 표준 Popover top layer에 올립니다. YouTube 선택자는 탐색을 빠르게 하는 힌트와 컨트롤 바 외형 보정에만 쓰며, 최종 적용 여부는 같은 공통 계약으로 판단합니다. 적용 뒤 두 프레임 안에 영상·컨트롤·경로가 안정되지 않거나 Popover가 닫히면 추가한 속성과 스타일을 제거해 원래 화면으로 복원합니다.

## Chrome에 설치하기

1. Chrome에서 `chrome://extensions`를 엽니다.
2. 오른쪽 위의 **개발자 모드**를 켭니다.
3. **압축해제된 확장 프로그램을 로드합니다**를 누릅니다.
4. 복제한 저장소의 루트 폴더를 선택합니다.
5. 이미 열어 둔 비디오 탭은 한 번 새로고침합니다.
6. 확장 아이콘을 고정하면 현재 비디오의 속도·화면 모드를 클릭으로도 바꿀 수 있습니다.

모든 일반 웹사이트에서 키를 처리하려면 `http`/`https` 사이트 접근 권한이 필요합니다. ViewTune은 네트워크 요청, 계정 정보 수집, 분석·추적을 하지 않으며 단축키, 목표 속도와 피드백 설정만 Chrome 동기화 저장소에 보관합니다.

## 업데이트와 문제 해결

- 소스 파일을 바꾼 뒤에는 `chrome://extensions`의 ViewTune 카드에서 **새로고침**을 누르고, 이미 열려 있던 비디오 탭도 새로고침해야 합니다. 확장만 다시 로드해도 기존 탭에는 이전 content script가 남을 수 있습니다.
- 팝업이 페이지의 이전 빌드를 감지하면 비디오 제어를 비활성화하고 **이 탭 새로고침**만 표시합니다.
- 검은 화면이나 예전 `W` 동작이 남으면 `chrome://extensions`에 ViewTune 카드가 둘 이상 켜져 있지 않은지 확인한 뒤 하나만 남기고 비디오 탭을 새로고침합니다.
- 현재 수정본은 `v0.3.0`, build `0.3.0-20260716-compat-i18n-v1`입니다. 빌드 정보는 업데이트 불일치가 감지될 때만 팝업에 표시됩니다.
- 확장 UI는 Chrome 언어에 따라 한국어 또는 영어로 표시됩니다.

## 제한 사항

- `<video>` 요소를 노출하지 않는 DRM·canvas 기반 플레이어는 제어할 수 없습니다.
- iframe 안의 영상도 속도와 B는 키보드로 제어할 수 있습니다. V는 iframe 내부 viewport만 채운 상태를 탭 전체 맞춤으로 오인하지 않도록 명시적으로 거절합니다.
- 일부 사이트가 재생 속도를 강제로 되돌릴 수 있습니다. ViewTune은 이를 계속 덮어쓰지 않습니다.
- Netflix에서는 안정적인 프레임 갱신을 위해 재생 중인 영상을 잠시 멈춘 뒤 속도를 적용하고 다시 재생하며, 지원 범위에 맞춰 최대 1.5×로 제한합니다. 다른 사이트의 목표 속도 설정 범위는 0.5×~4×입니다.
- V는 브라우저 기본 controls 또는 감지 가능한 HTML 컨트롤을 포함하고, 영상과 같은 프레임의 별도 조상을 안전한 host로 확인할 수 있는 플레이어만 확장합니다. canvas·Shadow DOM 내부에 컨트롤을 완전히 숨긴 사이트나 bare `<video>`처럼 안전한 host가 없는 구조에는 적용하지 않습니다.
- 이미 Popover로 쓰이는 요소를 빼앗거나, 플레이어와 겹치는 큰 형제 UI까지 함께 올려야 하거나, native fullscreen·Picture-in-Picture가 활성화된 경우 V를 안전하게 거절합니다. 속도 제어와 B는 각각의 조건 안에서 계속 사용할 수 있습니다.
- 문서 전체를 덮는 검은 overlay, 플레이어 DOM 재배치, 사이트별 `position: fixed` fallback은 사용하지 않습니다.

## 개발 확인

```powershell
npm run check
```

의존성이나 빌드 단계가 없는 Manifest V3 프로젝트입니다.

패키징 가능한 배포 ZIP을 만들려면 다음 명령을 실행합니다.

```powershell
npm run package
```
