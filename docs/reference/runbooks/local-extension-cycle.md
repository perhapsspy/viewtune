# 로컬 확장 개발 주기

소스 변경을 Chrome의 unpacked 확장에 반영하고, 이전 content script가 남아서 생기는 오판을 피하는 절차다.

## 준비 조건

- Chrome 114 이상
- Node.js 20 이상
- 저장소 루트에서 명령 실행
- `git status --short`로 사용자 작업과 현재 변경 범위 확인

## 최초 로드 또는 프로젝트 이동 후 복구

1. Chrome에서 `chrome://extensions`를 연다.
2. **개발자 모드**를 켠다.
3. 기존 ViewTune 카드가 현재 저장소 위치를 가리키는지 확인한다.
4. 카드가 사라졌거나 이전 폴더를 가리키면 **압축해제된 확장 프로그램을 로드합니다**에서 저장소 루트를 선택한다.
5. ViewTune 카드가 둘 이상 활성화되어 있으면 현재 저장소를 가리키는 하나만 남긴다. 중복 실행은 단축키 이중 처리와 이전 UI 재등장의 원인이 된다.
6. 확장 아이콘을 열어 하단 설치 버전이 `manifest.json`의 버전과 같은지 확인한다.

## 변경 반영 주기

1. 자동 검증을 실행한다.

   ```powershell
   npm run check
   ```

2. `chrome://extensions`의 ViewTune 카드에서 **새로고침**을 누른다.
3. 검사할 비디오 탭을 새로고침한다. 확장만 다시 로드하면 이미 열린 탭에는 이전 content script가 남을 수 있다.
4. 팝업에서 업데이트 불일치 안내 대신 현재 속도와 조작 버튼이 표시되는지 확인한다.
5. 설정을 바꾼 경우 팝업을 닫았다 다시 열어 단축키·목표 속도·화면 피드백 설정이 유지되는지 확인한다.

## 빠른 확인

- `[`·`]`·`R`·`G`가 현재 비디오 속도에 반영된다.
- `V`를 켠 뒤 `Esc`로 복원할 수 있다.
- `B`와 `V`의 활성 표시가 실제 화면 상태와 일치한다.
- 한글과 영문 입력 상태에서 같은 물리 키 위치가 동작한다.
- 입력창, 검색창, `contenteditable`에서는 ViewTune이 키를 가로채지 않는다.

## 복구와 중단 조건

- 팝업이 **탭 새로고침**을 요구하면 먼저 해당 탭을 새로고침한다. 계속되면 확장 카드의 설치 위치와 중복 카드를 확인한다.
- `Extension context invalidated` 오류가 남아 있으면 확장 오류 화면에서 기존 기록을 지운 뒤 ViewTune과 해당 비디오 탭을 차례로 새로고침한다. 이전 content script는 무효화 감지 후 자체 리스너를 해제하지만, 새 코드는 호스트 페이지를 새로고침해야 주입된다.
- `npm run check`가 실패하면 Chrome 수동 검사나 패키징으로 넘어가지 않는다.
- 검은 영상, 복원되지 않는 레이아웃, 사이트 단축키 침범이 발생하면 해당 상태와 재현 순서를 기록하고 [사이트 호환성 회귀 검사](compatibility-regression.md)에서 실패 범위를 좁힌다.
