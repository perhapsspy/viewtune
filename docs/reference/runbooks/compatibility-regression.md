# 사이트 호환성 회귀 검사

화면 모드나 재생 속도 구현을 바꾼 뒤 핵심 사이트와 일반 HTML5 비디오 계약이 유지되는지 확인하는 절차다.

## 준비 조건

- 먼저 [로컬 확장 개발 주기](local-extension-cycle.md)에 따라 확장과 검사할 탭을 모두 새로고침한다.
- 자동 검증 `npm run check`가 통과한 상태에서 시작한다.
- 각 검사 전에 비디오가 실제로 재생되고 있고, 입력창에 포커스가 없는지 확인한다.

## 공통 검사

각 지원 화면에서 다음을 확인한다.

1. `[`와 `]`를 눌러 속도가 0.5배 단위로 변한다.
2. `R`을 눌러 1배속으로 돌아온다.
3. `G`를 눌러 팝업에서 설정한 목표 속도로 바뀐다.
4. `V`를 켰다가 `Esc`로 복원한다. 같은 과정을 두 번 반복해 두 번째 전환도 깨지지 않는지 확인한다.
5. 충분히 넓은 플레이어 또는 전체화면에서 `B`를 켜 영상만 `cover`로 확대되는지 확인한다. 플레이어 크기와 전체화면 상태는 바뀌지 않아야 한다.
6. `B`와 `V`를 함께 켠 뒤 하나씩 꺼서 두 상태가 독립적으로 유지되는지 확인한다.
7. 팝업의 속도와 `B`·`V` 활성 표시가 실제 상태와 일치하는지 확인한다.

## 사이트별 매트릭스

| 대상 | 필수 화면 | 합격 기준 |
| --- | --- | --- |
| YouTube | 일반, `T` 영화관, 네이티브 전체화면 | `V`가 플레이어와 컨트롤을 함께 채우고 두 번 반복해도 원래 레이아웃으로 정확히 복원된다. `B`는 전체화면의 빈 영역을 영상 확대만으로 채운다. |
| 라프텔 | 재생 화면, 컨트롤 자동 숨김 상태 | `V`와 `Esc` 복원이 동작하고 컨트롤 표시 영역이 원래 크기에 남지 않는다. 속도 변경 후 영상이 계속 갱신된다. |
| 일반 HTML5 비디오 | 브라우저 기본 `controls`가 있는 페이지 | 속도 제어가 동작한다. 안전한 별도 플레이어 host를 확인할 수 있을 때만 `V`가 적용된다. |
| Netflix 도메인 | 로그인 여부와 무관한 Netflix 페이지 | ViewTune이 비디오·레이아웃·재생 속도를 바꾸지 않는다. 팝업은 조작 불가 상태를 조용히 표시하고 Netflix의 키 입력은 그대로 유지된다. |

## 실패 판정

다음 중 하나라도 발생하면 릴리스 검사를 중단한다.

- 영상은 검게 되고 자막이나 컨트롤만 보인다.
- `V`를 끈 뒤 플레이어 크기, 위치, 컨트롤 영역이 원래 상태로 돌아오지 않는다.
- 두 번째 `V` 전환에서 화면이 무너지거나 중복 레이어가 남는다.
- `B`가 플레이어 자체의 크기나 전체화면 상태를 바꾼다.
- 편집 영역이나 사이트 고유 단축키를 ViewTune이 가로챈다.
- Netflix에서 ViewTune이 컨트롤러나 페이지 이벤트를 활성화한다.

실패 시 사이트 DOM 이름을 바로 고정 선택자로 추가하지 않는다. 현재 `<video>`와 플레이어 표면의 기능·기하 조건, 적용 전 상태 보존, 실패 후 복원 경로를 먼저 확인한다.
